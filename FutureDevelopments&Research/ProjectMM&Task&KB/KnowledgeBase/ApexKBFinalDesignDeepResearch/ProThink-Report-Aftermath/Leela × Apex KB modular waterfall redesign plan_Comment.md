[Leela × Apex KB modular waterfall redesign plan](C:\\tmp\\leela-apex-kb-modular-waterfall-redesign-plan.md)

Core finding:

- The Apex KB skill already says: lock questions, use rankings to nominate sources, work one topic at a time, and continue only while questions remain unresolved.
- The Leela manual execution prompt introduced the problematic requirement to process all 197 sources for every topic and create complete ledgers/atlasses for all of them.
- Therefore, the explosion is primarily a Leela workflow override—not an unavoidable part of Apex KB.

The redesigned waterfall is:

```
Complete source custody
        ↓
One structural record per source
        ↓
Sparse topic/source relationships
        ↓
One query-specific source wave
        ↓
Material source analyses
        ↓
Minimal compiled knowledge
        ↓
Narrow independent acceptance
        ↓
Machine-only retrieval index
```

The report includes:

- confirmed problems and overlaps;
- verification against the live Apex KB skill and runtime;
- macro, meso, and micro workflows;
- proposed modular directory structures;
- single-owner rules preventing duplication;
- proposed 64 KB target and 128 KB hard browser-file limit;
- adaptive source waves instead of unsafe top-N selection;
- five OKRs with measurable results;
- seven implementation phases;
- a Skill Tree pilot plan;
- a validation matrix;
- explicit decisions required before implementation.

No Leela or Apex repository files were modified.
