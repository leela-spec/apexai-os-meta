## Where you might have a legitimate point

Two spots in Step 4 are under-specified and rely on implicit git usage without stating the command:

1. **Step 4.8 "Apply, confirm markers present"** — doesn't explicitly write `git apply <patch-file>` before the grep checks. It's implied by continuity from step 7, but an agent under ambiguity could substitute a non-git method (e.g., just reading and reasoning about the diff text instead of literally applying it).
2. **Step 4.9 "Commit the patch file only"** — doesn't spell out `git add <patch-file>` / `git commit -m "..."`. Same risk — the intent is git-native, but it's not written as literal commands.

Given this exact task history — where agents have repeatedly substituted shortcuts under ambiguity — leaving any step as narrative instead of literal command is a real weakness, even though "git is missing entirely" is not accurate.

## Fix

Step 4.8 and 4.9 should read:

```
8. Apply and verify:
   git apply <patch-file>
   grep -q "<required_marker>" <file>   (must find each required marker)
   grep -L "<forbidden_marker>" <file>  (must NOT find any forbidden marker)
   git checkout -- <file>

9. Commit patch artifact only:
   git add patch-packs/<NNN>-<name>.patch
   git commit -q -m "Add patch <NNN>"
```
