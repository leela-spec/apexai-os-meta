  Define the minimal repeatable process for using ChatGPT Agent Mode to create

  This guide intentionally avoids broad fallback frameworks and overengineered blockers.

core_distinction:
  agent_mode_builder:
    executor: "ChatGPT Agent Mode"
    role: "Creates validated patch artifacts from the live repo."
    autonomy: "bounded creative implementation within explicit target files"
    output: "patch pack artifacts only"
  codex_or_terminal_applier:
    executor: "Codex or terminal executor"
    role: "Applies and verifies existing patch artifacts."
    autonomy: "deterministic Git command execution"
    output: "implementation commit"

success_pattern:
  one_line: >
    Give Agent Mode a source-grounded semantic plan, exact target files,
    required/forbidden markers, and a Git-native artifact contract; let it work
    continuously until it creates one Git-generated patch per target file and
    validates every patch with git apply --check.
  why_it_worked:
    - "The live Git repository was the only source of truth."
    - "The source plan was specific enough to patch, not just strategic context."
    - "The target file list was explicit."
    - "The patch output list was explicit."
    - "The semantic contract was expressed as required and forbidden markers."
    - "Patches were generated by git diff, not hand-written by the model."
    - "Each patch touched exactly one target file."
    - "Target files were reverted after patch generation."
    - "The final builder state contained patch artifacts, not direct target-file changes."
    - "The applier later verified old blob matches, git apply, changed-file scope, and markers."

recommended_source_plan_quality:
  must_contain:
    clear_problem: "What is broken in the current repo/package behavior."
    clear_target_state: "What should be true after applying the patch pack."
    concrete_concepts: "Named concepts/sections/schema elements to add or change."
    non_goals: "What must not be touched or introduced."
    acceptance_markers: "Strings or observable conditions that prove the intended change landed."
  should_avoid:
    - "Long failed transcripts as primary input."
    - "Multiple competing plans with no authority order."
    - "Open-ended permission to improve anything."
    - "Broad repo discovery when target files are already known."
    - "Fallback engineering before the Git-native path is tried."

agent_mode_builder_process:
  - phase: "0_preflight"
    intent: "Confirm repo context without wasting the run on unnecessary blockers."
    commands:
      - "git rev-parse --show-toplevel"
      - "git rev-parse --is-inside-work-tree"
      - "git remote get-url origin"
      - "git branch --show-current"
      - "git status --porcelain"
    handling:
      expected_branch: "checkout requested branch if needed"
      missing_origin: "continue in artifact-export mode; do not claim pushed"
      dirty_unrelated_files: "record in manifest and continue if they do not overlap target_files or patch_pack_dir"
      dirty_target_or_patch_pack_files: >
        preserve a preexisting-dirty-state diff inside the patch-pack notes if practical,
        then restore task paths to HEAD for patch generation; report clearly in final report.
      fatal_only:
        - "not a Git worktree"
        - "source plan unreadable"
        - "target file missing"
        - "Git unavailable"

  - phase: "1_read_authority"
    intent: "Ground implementation in the latest source plan and live target files."
    actions:
      - "Read primary source plan first."
      - "Read optional supporting files only if available."
      - "Read every target file before editing."
      - "Create an internal target-change map; summarize it later in the manifest."

  - phase: "2_generate_one_patch_per_target"
    intent: "Create narrow Git-native patches."
    per_target_sequence:
      - "Restore target file to clean HEAD."
      - "Modify only that target file according to the source plan."
      - "Create patch with: git diff --no-ext-diff -- <target-file> > <patch-file>"
      - "Confirm patch file is non-empty."
      - "Confirm patch has exactly one diff --git header for the expected target."
      - "Revert target file to HEAD."
      - "Validate with: git apply --check <patch-file>"
    doctrine:
      - "Use git diff only."
      - "Do not hand-write unified diff hunks."
      - "Do not manually edit hunk headers."
      - "Do not run broad formatters."
      - "Do not normalize line endings across whole files."
      - "Do not combine unrelated files into one patch."

  - phase: "3_cumulative_validation"
    intent: "Prove the patch pack applies as a sequence."
    commands:
      - "apply-check every patch in order with git apply --check"
      - "apply every patch in order with git apply"
      - "git diff --name-only"
      - "search required markers"
      - "search forbidden markers"
      - "check forbidden paths are not changed"
    pass_definition:
      changed_files: "exactly equal target_files"
      required_markers: "present where applicable"
      forbidden_markers: "absent"
      forbidden_paths: "unchanged"

  - phase: "4_restore_builder_repo_state"
    intent: "Leave only patch-pack artifacts as the builder deliverable."
    commands:
      - "git checkout -- <target_files>"
      - "git status --porcelain"
    desired_state:
      target_files_modified: false
      patch_artifacts_present: true

  - phase: "5_write_patch_pack_metadata"
    artifacts:
      manifest:
        file: "<patch-pack-dir>/000-patch-manifest.md"
        include:
          - "repo"
          - "branch"
          - "source plan path"
          - "optional supporting files read"
          - "patch order"
          - "patch-to-target map"
          - "allowed outputs"
          - "forbidden paths"
          - "required markers"
          - "forbidden markers"
          - "validation commands run"
          - "validation result"
          - "dirty-state notes if any"
      apply_handoff:
        file: "<patch-pack-dir>/999-apply-patches.md"
        include:
          - "exact patch order"
          - "git apply --check before git apply"
          - "changed-file scope check"
          - "required/forbidden marker check"
          - "commit/push instruction"
          - "final report format"

  - phase: "6_commit_or_export"
    intent: "Make patch pack durable."
    preferred:
      - "git add <patch-pack-dir>"
      - "git commit -m '<patch-pack creation message>'"
      - "git push origin <branch>"
    fallback_when_push_unavailable:
      - "create downloadable archive"
      - "report origin/push limitation"
      - "do not claim repository push"

codex_or_terminal_apply_process:
  purpose: "Apply the already-created patch pack. This is separate from Agent Mode patch-pack creation."
  steps:
    - "Start from clean target branch."
    - "Run git apply --check for each patch in manifest order."
    - "Run git apply for each patch in manifest order."
    - "Verify git diff --name-only equals target_files."
    - "Verify required markers are present."
    - "Verify forbidden markers are absent."
    - "Commit and push implementation changes."
  default_rule: "Do not manually edit target files during apply."

overengineering_guard:
  target_behavior:
    - "Prefer the plain Git-native path."
    - "Keep recovery behavior small and explicit."
    - "Do not create custom fuzzy patch appliers by default."
    - "Do not add broad audit phases to a patch-pack builder run."
    - "Do not convert every warning into a run-stopping blocker."
  acceptable_minimal_safety:
    - "live repo source of truth"
    - "target file list"
    - "one patch per target"
    - "git diff patch generation"
    - "git apply --check validation"
    - "exact changed-file scope check"
    - "required/forbidden marker checks"
    - "artifact-only builder final state"

  <branch>
  Patch-pack path:
  <patch-pack-dir>

  Read first:
  <primary source plan>

    patch_file: <patch-pack-dir>/001-<name>.patch

  Allowed outputs:
  - <patch-pack-dir>/000-patch-manifest.md
  - <patch-pack-dir>/001-*.patch
  - <patch-pack-dir>/999-apply-patches.md

  Forbidden paths:
  - <path-or-glob>

  Required changes:
  - <semantic requirement>

  Required markers:
  - <marker>

  Forbidden markers:
  - <marker>

  Process:
  1. Preflight repo state.
  2. Read source plan and all target files.
  3. For each target file, edit only that file, generate the patch with git diff,
     revert the target, and validate the patch with git apply --check.
  4. Validate cumulative patch application.
  5. Verify exact changed-file scope and semantic markers.
  6. Revert target files.
  7. Commit/push patch artifacts, or export archive if push is unavailable.
  8. Return the final report schema.

short_rule: >
  The successful operation mode is bounded Agent Mode autonomy for patch-pack creation:
  source plan + exact target files + Git-generated one-file patches + git apply --check
  + cumulative scope/marker validation + artifact-only final state.
