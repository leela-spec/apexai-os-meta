SKILL.md Format Specification

Relevant source files
  • .gitignore
  • README.md
  • skills/skill-creator/references/schemas.md
  • spec/agent-skills-spec.md

This document defines the required structure and content of SKILL.md files, which serve as the entry point for all skills in the anthropics/skills repository. The specification covers YAML frontmatter fields, Markdown body requirements, naming conventions, and the validation rules enforced by the skills system.

For information about how skills are discovered and loaded by Claude, see page 2.1. For guidance on creating new skills using this format, see page 5.1. For running validation, see page 5.2.

Sources: README.md63-90 skills/skill-creator/references/schemas.md9-36

File Structure Overview

Every skill must be organized as a folder containing a SKILL.md file as its entry point. The file combines structured metadata (YAML frontmatter) with freeform instructions (Markdown body). Skills may also include bundled resources like scripts or assets.

YAML Frontmatter Specification

The YAML frontmatter must appear at the very beginning of SKILL.md, delimited by --- markers. It contains structured metadata that enables skill discovery and execution.

Required Fields

FieldTypeConstraintsPurpose
namestringkebab-case ([a-z0-9-]+)Unique identifier for the skill. Must match skill_name in evals.json. README.md87 skills/skill-creator/references/schemas.md30
descriptionstringComplete and clearDrives triggering. Describes what the skill does and when to use it. README.md88

Field Details:

  • name: A unique identifier for the skill (lowercase, hyphens for spaces). This name is used as a reference in the evals.json schema and for version tracking in history.json. README.md87 skills/skill-creator/references/schemas.md30-46
  • description: This is the primary triggering mechanism. It should provide a complete description of what the skill does and when to use it. README.md88-89

Optional Fields

FieldTypePurpose
licensestringClarifies usage rights (e.g., Apache-2.0 or Source-available). README.md20

Markdown Body Specification

The Markdown body follows the YAML frontmatter and contains the actual instructions, examples, and guidelines that Claude uses when the skill is active.

Progressive Disclosure System

Skills are designed to be loaded dynamically to improve performance on specialized tasks.

  1. Metadata (Frontmatter): Used for discovery and determining when to load the full skill. README.md6-11
  2. SKILL.md Body: Loaded into context only when the skill is active. Contains the primary instructions Claude will follow. README.md18 README.md75
  3. Bundled Resources: Scripts and resources referred to within the instructions that Claude can load as needed. README.md6

Content Conventions

  • Headings: Use clear H1 and H2 headers for organization. README.md73-81
  • Examples: Provide sample usage or "Example usage" lists to help Claude understand expected behavior. README.md77-79
  • Guidelines: List specific constraints or "Guideline" points for the task. README.md81-83

Validation and Evaluation Logic

The SKILL.md file is subject to automated validation against its associated evals.json file.

Natural Language to Code Entity Mapping: Evaluation Flow

This diagram shows how the SKILL.md content is tested using the evaluation system defined in the repository.

Evaluation Integration

  • Skill Matching: The name in SKILL.md frontmatter must match the skill_name field in evals.json. skills/skill-creator/references/schemas.md30
  • Functional Verification: The instructions in the Markdown body are tested against the expectations list in evals.json. skills/skill-creator/references/schemas.md20-23
  • Metric Tracking: Execution of the skill instructions is tracked in metrics.json, including tool calls like Read, Write, and Bash. skills/skill-creator/references/schemas.md163-183
