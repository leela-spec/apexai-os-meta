## Anti-sprawl controls

- **Control:** Scaffold files stay compact and point into appendices.
- **Control:** Candidate and evidence status remains explicit on every deep item.
- **Control:** No raw failure archive is copied into `MISTAKES.md`.
- **Control:** Templates provide forms, not hidden promptflow law.
- **Control:** `LEARNING_QUEUE.md` receives candidates only and cannot promote them.
- **Control:** If a source belongs mainly to another Special Ops agent, store only the placement implication here.

## Reusable failure patterns

### 1. Candidate-to-canon leak

- **Trigger:** A high-scoring candidate is copied into scaffold prose without status or validation boundary.
- **Impact:** Candidate material becomes de facto accepted truth.
- **Countermeasure:** Keep candidate status in `LEARNING_QUEUE.md` and appendices; scaffold only compact rules with promotion caveats.

### 2. Appendix bypass

- **Trigger:** Detailed source content enters `ESSENCE.md`, `BEST_PRACTICES.md`, `MISTAKES.md`, or `TEMPLATES.md`.
- **Impact:** Scaffold files become bloated and low-retrieval.
- **Countermeasure:** Move details to appendices and replace with one rule plus pointer.

### 3. Evidence overgeneralization

- **Trigger:** One failure/postmortem source is treated as universal doctrine.
- **Impact:** The KB hardens local incidents into global rules.
- **Countermeasure:** Mark postmortem-derived material as safeguard/evidence unless separately validated.

### 4. Hidden improvement application

- **Trigger:** A better idea appears while doing a bounded KB update.
- **Impact:** The run silently widens and mutates unauthorized surfaces.
- **Countermeasure:** Add the idea to `LEARNING_QUEUE.md` or an improvement section; do not apply it in the current run.

### 5. Connector replacement masquerade

- **Trigger:** The executor lacks native `git apply` and uses the GitHub connector to replace whole files anyway.
- **Impact:** The changed-file set, diff validity, and patch semantics are not proven.
- **Countermeasure:** stop before write execution; hand off exact unified diffs to Codex with zero content-generation freedom.

### 6. Legacy promptflow basis leak

- **Trigger:** old KB promptflow references remain in scaffold or manifest source-basis sections.
- **Impact:** corrected rules can be overridden by historical execution assumptions.
- **Countermeasure:** mark old promptflows historical-only and use corrected promptflow as the sole current update authority.
