# Process Blueprint System

This file is a companion process map. Runtime-governing process law lives in managed canon and managed rituals; this file only points to those surfaces.

Use the following map:

- `managed/rules/OPERATING_SPINE_CANON.md` — top-level operating order and control posture
- `managed/rituals/BOOTSTRAP.md` — startup checks and bounded entry
- `managed/rituals/SESSION_EXPORT_PROTOCOL.md` — session trace protocol
- `managed/rituals/NIGHT_PLANNING_PROTOCOL.md` — cross-session synthesis and planning ritual
- `managed/rituals/CHECKLISTS.md` — lightweight recurring validation and safety checks
- `managed/rules/QA_HYGIENE_PROTOCOL.md` — hygiene findings, severities, and routing consequences
- `managed/rules/PROMOTION_PROTOCOL.md` — truth-change gate
- `docs/PROJECT_ROUTING.md` — companion routing examples
- `docs/ROLE_SYSTEM.md` — companion role and handoff examples

Use this file as a map for where process authority lives, not as a substitute for that authority.
