## Macro Synthesis

Migration should preserve source-grounded patterns, not copy old runtime assumptions. The old system is useful as evidence for role boundaries, routing, validation, and scaffold design; it is not itself the current Apex runtime map.

## Meso Synthesis

Migration decisions should pass three filters:

1. Is the item a reusable pattern or merely an old implementation detail?
2. Does the item require current repo verification before use?
3. Should the item become skill text, a workflow, a lint rule, a wiki page, or remain historical evidence?
