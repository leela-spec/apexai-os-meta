# Agent Doctrine and Promotion Patterns

## Definition

A reusable pattern family for durable agent or capability-lane knowledge. It combines a five-file scaffold, compact role-boundary surfaces, candidate-only learning queues, owner/validator separation, scaffold/appendix split, and phase-gated promotion from candidate material to accepted doctrine.

## Operating Rules

```yaml
rules:
  - "Keep compact activation and boundary doctrine in a small orientation surface."
  - "Store detailed evidence, ledgers, examples, and QA traces in appendices or lower-level wiki pages."
  - "Treat learning queues as candidate material only."
  - "Separate source material, candidate material, accepted doctrine, and runtime truth."
  - "Use owner and validator routing before promoting candidate material."
  - "Design wiki chunks so they remain useful when retrieved in isolation."
```
