# Operator Approval Note

This note applies to KB lifecycle runs under `apex-meta/kb/`.

For future KB runs, the approval checkpoint should be interpreted by clear operator intent, not by one mandatory literal wording.

Accepted approval must be explicit, written by the operator, and scoped to the current KB unless the operator clearly expands the scope.

Do not treat vague acknowledgement, discussion, analysis requests, or status questions as approval.

The purpose of the checkpoint is consent for the next lifecycle phase, not exact-word enforcement.
