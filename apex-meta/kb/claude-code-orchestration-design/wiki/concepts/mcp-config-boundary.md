# MCP Config Boundary

```yaml
pattern: "MCP configuration is a trust-sensitive runtime boundary and must not be generated by Phase 2 wiki compile."
used_when:
  - "A future design asks whether to add or commit MCP server configuration."
not_used_when:
  - "The task only needs internal KB retrieval, local files, or deterministic scripts."
reads:
  - "server purpose"
  - "scope: user/project/managed"
  - "data access surface"
  - "trust and injection-risk assessment"
writes:
  - "deferred policy record"
  - "review checklist"
  - "never writes .mcp.json during S6"
token_efficiency:
  - "Keeps MCP as a late-loaded decision topic rather than always-on orchestration context."
drift_controls:
  - "Requires explicit allowlist and operator approval before any committed config."
unresolved_or_deferred:
  - "MCP server allowlist."
  - "Committed config path and ownership."
  - "Injection-risk validation procedure."
```
