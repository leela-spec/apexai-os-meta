# Field Schema

The following table defines the canonical fields for Apex task files. Each field is marked as **required** (must be provided by the author), **optional** (may be omitted), or **computed** (generated by the system).

## Field Descriptions

- **name** – short title describing the task.
- **status** – current state of the task using the H1 enum values (open, in‑progress, blocked, done, deferred).
- **created** / **updated** – ISO‑8601 timestamps; these are automatically recorded when the task is created or edited.
- **priority** – one of high, medium, or low; weights of 3/2/1 respectively.
- **urgency** – computed from `due_date` as the number of days until the due date; omitted if no due date is set.
- **depends_on** – list of integer task IDs; the task becomes actionable only when all dependencies have `status: done`.
- **parallel** – boolean indicating that the task can run concurrently with its dependencies; defined in the dependency structure.
- **conflicts_with** – list of task IDs that cannot be executed concurrently.
- **effort** – qualitative estimate of size or complexity: XS, S, M, L, or XL.
- **epic** – slug or identifier of the parent epic (if any).
- **due_date** – ISO‑8601 date by which the task should be completed.
- **acceptance_criteria** – bullet list defining conditions that must be met for the task to be accepted.
- **definition_of_done** – bullet list specifying objective completion indicators.
