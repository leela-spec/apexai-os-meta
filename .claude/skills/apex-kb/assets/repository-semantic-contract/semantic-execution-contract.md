# Semantic Execution Contract

Before selecting any source, run the intake and intent lock. In one conversational pass, capture from the operator (never assume): the operator's intent, the KB identity ("this KB is about X"), where the real source material lives and what is out of scope, and what success means for this KB. Validate each locked topic against the KB's own scope evidence — its path, sibling topics, and a light filename sample (never the KB's own freshly-written `kb-schema.md`/`README.md` or generated output, which are circular) — and read your whole understanding back to the operator, including your recommended output tier and corpus breadth (default narrow) with a one-line reason each. Get the operator's explicit confirmation and record it in `manifests/run-intent.md` (an allowed complete whole-file write) with `operator_confirmed: true` and the verbatim affirmative. Do not select sources, write Phase 1/Phase 2 files, or commit until that confirmation exists. A topic whose terms have zero correspondence to the KB's scope evidence is a topic-lock mismatch to confirm with the operator, not a source-access blocker to write up.

Lock target questions and answer requirements before selecting sources. Read actual raw sources, using existing Phase 0 rankings only as candidate navigation. Continue until critical/routine questions are covered or evidence is genuinely unavailable. Create complete Phase 1 and Phase 2 files, reread stored files, preserve contradictions, and manually review claims and pointers. Stop only affected topics when evidence cannot be read. Report `compiled_unvalidated` only after independent clean-context semantic acceptance; otherwise report `partial` or `analysis_complete_unvalidated`.

Do not run Python or shell commands. Do not scaffold, intake sources, hash files, modify manifests, regenerate Phase 0, rebuild indexes/retrieval, run lint/quality/postflight, or claim `query_ready`.

Availability of an account-level ChatGPT Skill is optional. A semantic execution must not stop merely because an installed Skill cannot be invoked. The repository-local files under `semantic-contract/` are sufficient authority for Phase 1 and Phase 2 semantic work.

A known readable canonical source that can answer an unresolved critical/routine question must be read; it cannot be demoted to a reopen trigger. Unopened sources are candidates, never evidence. Connector constraints may force `partial`, never a lower completion standard.
