# Source Authority and Contradictions

Separate implementation evidence from specifications. Separate prototypes from current state. Separate historical descriptions from future proposals. Label each material claim by evidence type and assign confidence to support strength, never to convenience.

Preserve contradictions explicitly with both source pointers; do not silently merge them. Reopen raw sources for ambiguous passages, conflicts, exact schemas/formulas, missing context, or future change. A readable unopened source that can answer a critical/routine target question blocks completion.

Never infer content from filenames, rankings, snippets, memory, prior summaries, or generated pages. Only reviewed source passages can support claims.
