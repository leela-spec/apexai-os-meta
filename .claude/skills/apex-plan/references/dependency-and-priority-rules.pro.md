# Dependency and Priority Rules## PurposeDefine how `apex-plan` proposes dependency links, priority, due dates, and provisional focus candidates. These rules are qualitative planning rules. Exact graph eligibility, urgency scoring, next-task ranking, blocker scanning, and registry-derived computation belong to `apex-sync`.## Dependency field```yamldependency_contract:  canonical_field: depends_on  field_type: integer_array  example: [1, 3]  meaning: this_task_should_not_start_until_all_listed_task_numbers_are_done  exact_eligibility_rule_owner: apex-sync  planning_owner: apex-plan```## Dependency proposal rules```yamldependency_proposal_rules:  propose_depends_on_when:    - one_task_requires_the_output_of_another_task    - one_task_requires_a_decision_created_by_another_task    - one_task_requires_review_or_approval_from_another_task    - one_task_would_be_wasteful_or_invalid_before_another_task_finishes  avoid_depends_on_when:    - order_is_only_a_preference    - tasks_can_proceed_in_parallel    - dependency_is_based_on_guesswork    - dependency_would_create_unnecessary_serial_work  confidence_labels:    high: directly_supported_by_supplied_context    medium: plausible_from_task_relationships_but_operator_should_review    low: weak_or_ambiguous_relationship```## Dependency validation rules`apex-plan` may reason about dependency quality, but must not run deterministic graph validation.```yamlplanning_validation:  check_for_obvious_self_dependency: true  check_for_obvious_duplicate_dependency: true  flag_possible_cycle_for_operator_review: true  do_not_run_cycle_detection_script: true  do_not_compute_actionability: true  do_not_compute_unlock_depth: true```## Parallel and conflict rules```yamlparallel_rules:  true_when:    - tasks_have_no_known_depends_on_relationship    - tasks_produce_independent_outputs    - tasks_can_be_reviewed_separately  false_when:    - tasks_share_a_required_sequence    - tasks_touch_the_same_deliverable_in_conflicting_ways    - one_task_is_a_review_gate_for_the_other  unknown_when:    - context_is_insufficientconflicts_with_rules:  use_when:    - two_tasks_should_not_be_executed_at_the_same_time    - they_modify_the_same_planning_artifact_or_output    - they_require_mutually_exclusive decisions  do_not_discover_by_repo_scan: true```## Priority policy```yamlpriority_policy:  canonical_field: priority  allowed_values:    - high    - medium    - low  deterministic_weights:    high: delegated_to_apex_sync    medium: delegated_to_apex_sync    low: delegated_to_apex_sync  apex_plan_role:    - assign_proposed_priority    - explain_rationale    - mark_review_needed_when_uncertain  apex_plan_must_not:    - compute_final_rank_score    - compute_exact_next_task    - override_operator_priority```## Priority assignment guidance```yamlpriority_assignment_guidance:  high:    use_when:      - task_is_required_for_core_project_progress      - task_unblocks_multiple_later_tasks      - task_resolves_a_major_design_or_delivery_uncertainty      - operator_explicitly_marks_it_as_critical  medium:    use_when:      - task_matters_but_does_not_define_near_term_success      - task_supports_a_high_priority_epic      - priority_is_unclear_and_operator_review_is_needed  low:    use_when:      - task_is_cleanup_or_optional_improvement      - task_can_wait_without_blocking_core_progress      - task_is_a_nice_to_have_or_future_extension```## Urgency and due-date policy```yamlurgency_policy:  canonical_field: due_date  allowed_values:    - ISO8601_date    - null  urgency_score_owner: apex-sync  apex_plan_role:    - preserve_operator_supplied_due_dates    - propose_due_date_when_operator_context_clearly_implies_one    - set_null_when_no_real_due_date_is_known    - explain_time_sensitivity_in_words  apex_plan_must_not:    - compute_days_until_due    - compute_numeric_urgency_score    - invent_due_dates```## Provisional focus candidates`apex-plan` may propose focus candidates, but the output must not be framed as exact next-task computation.```yamlfocus_candidate_rules:  allowed_output_name: provisional_focus_candidates  allowed_basis:    - operator_goal    - dependency_proposal    - priority_rationale    - due_date_rationale    - success_criteria    - known_constraints_supplied_by_operator  required_label: provisional_not_exact_next_task  forbidden_basis:    - unseen_registry_state    - blocker_scan    - stale_detection    - exact_dependency_eligibility_script    - urgency_score_script    - exact_next_task_algorithm  handoff_if_exact_ranking_needed:    package: apex-sync    reason: exact_ranking_requires_deterministic_computation```## Output language patternsUse:```text"Provisional focus candidate""Priority proposal""Due-date proposal""Dependency proposal""Needs operator review""Exact computation delegated to apex-sync"```Do not use:```text"The next task is...""Blocker scan shows...""Registry says...""Exact rank is...""Urgency score is...""Task is actionable"```
