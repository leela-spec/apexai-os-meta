# Operator Gate## PurposeDefine the approval gate for `apex-plan`. The planning packet is not applied state. The operator must approve, revise, reject, split, or defer before any downstream mutation can occur.## Gate principle```yamloperator_gate_principle:  apex_plan_output_is_proposal: true  mutation_requires_operator_approval: true  mutation_owner: apex-session  exact_computation_owner: apex-sync  no_implicit_apply: true```## Gate statuses```yamloperator_gate_statuses:  operator_review_needed:    meaning: packet_is_ready_for_operator_review_but_not_approved    downstream_mutation_allowed: false  approved:    meaning: operator_accepts_packet_as_basis_for_downstream_action    downstream_mutation_allowed: true    downstream_owner: apex-session  needs_revision:    meaning: operator_wants_changes_before_approval    downstream_mutation_allowed: false  rejected:    meaning: operator_rejects_packet    downstream_mutation_allowed: false  split_needed:    meaning: packet_scope_is_too_large_or_mixed    downstream_mutation_allowed: false  deferred:    meaning: packet_may_be_revisited_later    downstream_mutation_allowed: false```## Required gate blockEvery `apex_plan_packet` must include:```yamloperator_gate:  status: operator_review_needed  requested_operator_action: approve | revise | reject | split | defer  approval_required_before_mutation: true  mutation_owner_if_approved: apex-session  exact_computation_owner_if_needed: apex-sync  operator_questions:    - question: string      reason: string```## Presentation rulesWhen presenting the packet:1. Show the project capture first.2. Show epic drafts before task drafts.3. Show dependency and priority proposals after task drafts.4. Show provisional focus candidates only after explaining that they are not exact next-task computation.5. Show the downstream handoff section.6. End with the operator gate and requested action.## Revision behavior```yamlrevision_behavior:  if_operator_revises_goal:    update_project_capture: true    recheck_epics_and_tasks: true  if_operator_reorders_tasks:    preserve_operator_order_as_planning_preference: true    do_not_convert_to_exact_rank: true  if_operator_changes_priority:    accept_operator_priority: true    mark_source: operator_override  if_operator_adds_due_date:    preserve_due_date: true    do_not_compute_urgency_score: true  if_operator_approves:    do_not_write_files_in_apex_plan: true    produce_apex_session_handoff_instruction: true```## Approved handoff languageIf the operator approves, use language like:```textApproved planning packet is ready for `apex-session` to apply as durable task files or session records. Exact dependency eligibility, urgency scoring, blocker scans, registry rebuilds, or next-task computation should be run through `apex-sync` only if needed.```## Non-approval languageIf the operator has not approved, use language like:```textThis is still a planning proposal. No state has been changed. Choose approve, revise, reject, split, or defer.```## Mutation safety rules```yamlmutation_safety:  apex_plan_must_not:    - create_files    - update_files    - delete_files    - change_task_status    - write_handoff_files    - rebuild_registry    - apply_operator_approval_itself  downstream_mutation_path:    1_operator_approves_packet: true    2_apex_session_prepares_mutation_delta: true    3_operator_confirms_mutation_delta: true    4_apex_session_applies_mutation_if_allowed: true```## Gate failure modes```yamlgate_failure_modes:  hidden_mutation:    failure: response_implies_state_was_changed    correction: explicitly_state_no_state_changed_and_route_to_apex_session  exact_rank_claim:    failure: response_claims_exact_next_task    correction: relabel_as_provisional_or_route_to_apex_sync  missing_review_flag:    failure: packet_has_no_operator_gate    correction: add_operator_gate_before_completion  invented_source:    failure: packet_claims_unread_source_or_unscanned_repo_state    correction: remove_claim_and_mark_input_missing_or_assumed```
