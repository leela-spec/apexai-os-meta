# Task Decomposition Rules## PurposeDefine how `apex-plan` converts project intent into epics and task drafts. These are planning rules only. They produce proposals for operator review and do not create files.## Decomposition model```yamldecomposition_model:  hierarchy:    - project    - epic    - task    - acceptance_criteria    - definition_of_done  output_level:    normal_case: epic_plus_task_drafts    small_scope_case: task_drafts_only    unclear_scope_case: project_capture_plus_open_questions  numbering:    epic_numbering: sequential_in_packet    task_numbering: sequential_in_packet    numbering_is_provisional_until_apex_session_applies_mutation: true```## Project capture rules1. Convert the operator’s request into a project goal in one or two sentences.2. Separate goals from non-goals.3. Extract success criteria when present.4. Convert constraints into planning constraints, not blockers discovered by scanning.5. Mark missing information instead of inventing it.6. Preserve operator wording when it defines intent, acceptance criteria, or scope boundaries.## Epic rulesAn epic should exist when a group of tasks shares one deliverable or one planning outcome.```yamlepic_rules:  create_epic_when:    - multiple_tasks_share_one_outcome    - a_deliverable_needs_separate_acceptance_criteria    - work_can_be_reviewed_as_a_chunk    - dependencies_are_easier_to_explain_at_chunk_level  avoid_epic_when:    - request_is_one_small_task    - epic_would_only_rename_the_project    - operator_asked_for_a_minimal_task_list  each_epic_requires:    - epic_title    - epic_goal    - deliverable    - success_criteria    - non_goals_if_any    - operator_review_flags_if_any```## Task splitting rules```yamltask_splitting_rules:  each_task_should_have:    - one_clear_objective    - one_expected_output    - acceptance_criteria    - definition_of_done    - dependency_field    - priority_proposal    - due_date_proposal    - effort_estimate_if_possible  split_task_when:    - it_has_multiple_outputs    - it_requires_separate_review_gates    - it_mixes_research_and_implementation    - it_mixes_planning_and_mutation    - it_has_independent_dependencies    - it_is_too_large_for_one_focused_work_session  keep_task_together_when:    - splitting_would_create_tiny_non_actionable_fragments    - all_steps_share_the_same_output_and_review_gate    - dependencies_are_identical```## Task draft templateUse this shape inside `apex_plan_packet.task_drafts`:```yamltask_draft:  task_number: integer  epic_number: integer  title: concise_action_title  status: open  description: what_the_task_does  expected_output: concrete_artifact_or_decision  acceptance_criteria:    - observable_condition_for_acceptance  definition_of_done:    - completion_condition  priority: high | medium | low  due_date: ISO8601 | null  depends_on: []  parallel: true | false | unknown  conflicts_with: []  effort: XS | S | M | L | XL | unknown  source_note: where_the_task_came_from  operator_review_flags: []```## Acceptance criteria rules```yamlacceptance_criteria_rules:  good_acceptance_criteria:    - observable    - testable_by_operator_or_reviewer    - tied_to_expected_output    - specific_enough_to_prevent_vague_completion  avoid:    - "looks good"    - "finish task"    - "improve it"    - acceptance_criteria_that_require_unavailable_external_state```## Definition of done rules```yamldefinition_of_done_rules:  purpose: define_when_the_task_can_be_marked_done_after_execution  must_be:    - concrete    - reviewable    - compatible_with_H1_status_done  should_not:    - require_apex_plan_to_mark_done    - require_apex_plan_to_update_files    - require_apex_plan_to_run_validation_scripts```## Parallel and conflict proposal rules```yamlparallel_conflict_rules:  parallel:    true_when: tasks_can_be_worked_independently_after_dependencies_are_satisfied    false_when: tasks_share_a_required_sequence_or_review_gate    unknown_when: evidence_is_insufficient  conflicts_with:    use_when: two_tasks_touch_the_same_output_or_cannot_safely_progress_together    leave_empty_when: no_conflict_is_known_from_supplied_context    do_not_scan_repo_to_discover_conflicts: true```## Ambiguity handling```yamlambiguity_handling:  if_project_goal_unclear:    output_project_capture_plus_questions: true    do_not_force_full_decomposition: true  if_task_order_unclear:    propose_low_confidence_dependency_options: true    mark_operator_review_needed: true  if_priority_unclear:    set_priority_to_medium_with_review_flag: true  if_due_date_unclear:    set_due_date_to_null: true  if_scope_too_large:    propose_epic_split: true    mark_split_needed_as_operator_gate_option: true```## Forbidden decomposition behavior```yamlforbidden:  - do_not_create_or_update_task_files  - do_not_assign_final_task_numbers_as_repo_truth  - do_not_compute_exact_next_task  - do_not_scan_repo_for_missing_tasks  - do_not_rebuild_registry  - do_not_mark_tasks_done  - do_not_write_handoff_files```
