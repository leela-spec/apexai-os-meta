The skill converts operator intent, rough notes, existing project/task excerpts, or planning context into an `apex_plan_packet` for operator review. It proposes epics, task drafts, dependency links, priority, due dates, and provisional focus rationale. It does not run scripts, compute the exact next task, scan blockers, detect stale state, rebuild registries, write handoff files, or mutate repo state. Any exact computation must be handed to `apex-sync`; any durable write or session-continuity update must be handed to `apex-session`.## Skill Contract```yamlskill_contract:  skill_name: apex-plan  cluster: A_PLAN  implementation_type: pure_skill_reasoning_no_scripts  primary_output: apex_plan_packet  output_mode: present_in_chat_for_operator_review  mutation_allowed: false  script_execution_allowed: false  owns:    - If the operator is asking for exact computation, blocker scanning, stale detection, registry rebuild, file writes, or handoff-file creation, do not perform that work; route it to the correct downstream package in the final handoff.2. **Phase 2 — Collect the planning basis.** Use only the context supplied in the current request and any explicitly provided readable project/task excerpts. Mark missing inputs as `missing`, `partial`, or `operator_review_needed`; do not invent source facts.3. **Phase 3 — Normalize the project capture.** Convert the operator’s intent into a concise project goal, scope, non-goals, success criteria, constraints, assumptions, and open questions.4. **Phase 4 — Draft epics.** Group work into outcome-oriented epics. Each epic must have a short title, goal, deliverable, success criteria, and reason for existing.5. **Phase 5 — Draft tasks.** Break each epic into task drafts using `references/task-decomposition-rules.md`. Each task draft must include title, description, expected output, acceptance criteria, definition of done, status proposal, priority proposal, due-date proposal, and review flags.6. **Phase 6 — Propose dependency structure.** Use `depends_on` as the dependency field. Propose only dependency links that can be justified from the planning basis. Do not compute exact eligibility or unlock depth.7. **Phase 7 — Propose priority and due date.** Use `priority: high | medium | low` and `due_date: ISO8601 | null`. Explain the rationale in plain language. Do not calculate urgency score; delegate exact score calculation to `apex-sync`.8. **Phase 8 — Draft provisional focus candidates.** Provide a short ranked or grouped set of candidate next focus areas with rationale. Label the result as provisional planning guidance, not exact next-task computation.9. **Phase 9 — Run boundary validation.** Check that the draft does not claim to have run scripts, written files, scanned blockers, rebuilt registries, detected stale state, computed exact next task, or created handoff files.10. **Phase 10 — Present the operator gate.** Present the `apex_plan_packet` and ask the operator to approve, revise, reject, split, or defer. Do not present the packet as applied state.11. **Completion gate.** The skill is complete only when it has produced an `apex_plan_packet` with `operator_gate.status: operator_review_needed` or a clearly labeled downstream handoff explaining which parts belong to `apex-sync` or `apex-session`.## Refusal and reroute rules```yamlreroute_rules:  if_asked_to_run_scripts:    response: "Do not run scripts in apex-plan.
