## Failure Mode Table

|Failure mode|Trigger|Correction behavior|
|---|---|---|
|`missing_next_day_plan`|No `next_day_plan` is created or defined.|Create a `next_day_plan`; if context is absent, use `bootstrap_mode` and mark `low_confidence_auto_generated`.|
|`missing_fixed_flow`|One of F1, F2, F3, or F4 is absent without omission reason.|Add the missing flow or explicitly omit it with reason and recovery logic.|
|`represented_flow_missing_packet`|A planned or compressed flow has no `flow_packet`.|Create or define a `flow_packet` for that flow.|
|`represented_flow_missing_prompt_pack_reference`|A represented flow has no `prompt_pack_reference`.|Add a reference to the per-flow `flow_prompt_pack`; use degraded generic mode if prompt context is missing.|
|`prompt_pack_not_copy_paste_ready`|A prompt pack lacks finalized copy-paste prompts.|Finalize prompt bodies or mark prompt pack as `generic_degraded_mode` with review flag.|
|`missing_prompt_rationale`|Provider or prompt design rationale is missing.|Add short `provider_rationale` and `prompt_design_rationale`.|
|`missing_prompt_hints`|Prompt failure or learning hints are missing.|Add small `prompt_failure_hints` and `prompt_learning_hints`.|
|`missing_usage_tracking_tags`|Usage tags are absent from a represented flow or prompt pack.|Add specific tags when usage context exists; otherwise add generic tags and usage review flag.|
|`unsafe_calendar_write`|Calendar write/update is presented as completed without operator acceptance.|Convert to pending request or write failure note; do not claim write completion.|
|`missing_FlowRecap_handoff`|Represented flow lacks `FlowRecap_handoff_block`.|Add handoff block with raw dump expectations, completion states, and operator validation question.|
|`missing_input_hard_block`|Missing input stops output generation.|Continue with best available context and record missing input as `operator_review_flag`.|
|`project_execution_drift`|Output begins doing project work instead of planning it.|Remove execution content and return to plan, packets, prompts, requests, and handoffs.|
|`FlowRecap_execution_drift`|Output creates recap packet or runs FlowRecap.|Replace recap content with `FlowRecap_handoff_block`.|
|`status_merge_drift`|Output merges status or creates APSU/status-merge result.|Remove merge content and preserve only next-day planning outputs.|
