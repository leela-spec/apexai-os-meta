## Artifact-production process map

For file, patch, KB, prompt, and workflow-production runs, the process order is:

1. name the bounded target artifact and exact target path or file group
2. create the requested artifact or identify the concrete blocker that prevents creation
3. validate the concrete artifact against scope, source fit, format, and mechanical checks
4. repair once when validation fails and a bounded repair is safe
5. stop for operator validation before widening scope or moving to the next unit
6. record learning only after the artifact result and validation state are known

This map does not override the operating spine. It clarifies the lower-level file-production loop inside an already-authorized artifact-manufacturing run.

Use this file as a map for where process authority lives, not as a substitute for that authority.
