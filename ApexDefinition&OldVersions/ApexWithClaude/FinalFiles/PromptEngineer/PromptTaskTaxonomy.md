## Task Selection Rules

```
task_selection_rules:  priority_order:    1: explicit_operator_task_type    2: requested_output_shape    3: dominant_action_verb    4: source_context_shape    5: workflow_context_hint    6: likely_downstream_use  decision_rules:    explicit_operator_task_type:      rule: "If the operator names a valid prompt_task_type, use it unless it conflicts with the requested output."      conflict_behavior: "Mark operator_review_recommended and list the conflict."    requested_output_shape:      rule: "If the operator names the final artifact shape, infer task type from the output shape."      examples:        - verbs: ["improve", "tighten", "revise prompt"]          likely_task_type: prompt_refinement    source_context_shape:      rule: "Classify based on what material is available."      examples:        - source_shape: existing_prompt          likely_task_type: prompt_refinement        - source_shape: long_document_set          likely_task_type: long_context_digestion        - source_shape: messy_process_notes          likely_task_type: workflow_extraction        - ## Provider-Neutral Task Patterns

```
provider_neutral_task_patterns:  universal_prompt_parts:    required:      - role      - task      - context      - constraints      - output_contract      - validation_criteria      - stop_condition    optional:      - assumptions      - examples      - failure_modes      - follow_up_instruction      - light_capture_hint  task_patterns:    research:      pattern:        role: "You are a careful researcher."        task: "Find or verify the requested information."        context: "Use only the supplied material unless web/current search is explicitly available."        output_contract: task: "Convert the goal and constraints into an ordered plan."        context: "Use supplied project state and operator intent."        output_contract: "Return phases, dependencies, risks, gates, and next actions."        validation_criteria: "Make sequence and tradeoffs explicit."    coding:      pattern:        role: "You are a technical implementation assistant."        task: "Solve or prepare validation_criteria: "Avoid unrelated rewrites."    critique:      pattern:        role: "You are a strict reviewer."        task: "Evaluate the artifact against the supplied criteria."    file_generation:      pattern:        role: "You are a file author."        task: "Create exactly one complete final file."        context: "Use the supplied file path, contract, and prior        task: "Extract repeatable stages, inputs, outputs, gates, and roles."        context: "Use messy process material as source."        output_contract: "Return normalized workflow record ## Ambiguity Rules

```
task_classification_ambiguity_rules:  ambiguity_detection:    unclear_task_type:      trigger: "More than one prompt_task_type fits and no dominant deliverable is explicit."      correction: "Select the safest primary task type, list secondary candidates, and mark operator_review_recommended."    missing_expected_output:      trigger: "The operator asks for work but does not define the output shape."      correction: "Infer expected output from task type and mark expected_output_type as low_confidence_auto_generated."    conflicting_action_and_output:      trigger: "The action verb implies one task type but the requested output implies another."      correction: "Prioritize requested output shape and flag the conflict."    source_shape_conflict:      trigger: "Source material suggests review/refinement, but operator wording suggests generation from scratch."      correction: "Treat the source artifact as authoritative context and flag whether rewriting is intended."    workflow_prompt_boundary_unclear:      trigger: "The task asks to define workflow/process labels and write prompt content in the same step."      correction: "Classify prompt_task_type from the deliverable, but request workflow-process-design validation for labels."    provider_style_boundary_unclear:      trigger: "The task asks for provider-specific optimization while no provider is chosen."      correction: "Use provider_unspecified and add operator_review_flag missing_provider_target."  default_resolution:    when_in_doubt:      primary_rule: "Choose the task type that matches the final deliverable."      secondary_rule: "Prefer validation or critique when an existing artifact plus rules are supplied."      fallback_rule: "Use operator_defined only when taxonomy coverage is genuinely insufficient."
```

## Operator Review Flags

```
task_classification_operator_review_flags:  allowed:    - unclear_prompt_task_type    - multiple_valid_task_types    - missing_expected_output_type    - conflicting_action_and_output    - missing_provider_target    - workflow_labels_need_external_validation    - process_stage_unclear    - source_context_too_thin    - operator_defined_type_used    - likely_wrong_granularity    - output_shape_not_supported_by_context  flag_rules:    unclear_prompt_task_type:      use_when: "No single task type dominates."      recommended_response: "List the likely primary and secondary task types."    multiple_valid_task_types:      use_when: "The task legitimately combines multiple prompt functions."      recommended_response: "Choose one primary type and record secondary_task_types."    missing_expected_output_type:      use_when: "Output shape must be inferred."      recommended_response: "Infer conservatively and ask for operator correction only if critical."    conflicting_action_and_output:      use_when: "The requested verb and requested artifact imply different task types."      recommended_response: "Prioritize artifact shape and flag the mismatch."    missing_provider_target:      use_when: "Provider-specific style would matter but no provider is selected."      recommended_response: "Use provider_unspecified until routing is available."    workflow_labels_need_external_validation:      use_when: "Workflow or process labels are needed but not clearly supplied."      recommended_response: "Request workflow-process-design validation."    source_context_too_thin:      use_when: "The task can proceed only in low-confidence mode."      recommended_response: "Generate best effort and mark low_confidence_auto_generated."    likely_wrong_granularity:      use_when: "The task seems too broad for one prompt packet."      recommended_response: "Suggest splitting into prompt_sequence or smaller prompt_packet units."
```
