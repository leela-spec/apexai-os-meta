## Failure Modes

```
loop_failure_modes:  unclear_loop_selection:    trigger: "Multiple loops fit and no dominant output or source shape is clear."    correction: "Choose the safest loop for the final deliverable and mark operator_review_recommended."  missing_validation_criteria:    trigger: "The selected loop requires critique or validation but no criteria are supplied or inferable."    correction: "Use generic output-contract checks and flag missing_validation_criteria."  loop_scope_creep:    trigger: "The loop starts producing unrelated artifacts or additional files."    correction: "Return to the selected loop phase and remove unrelated outputs."  infinite_revision_cycle:    trigger: "The critique/revise cycle keeps generating new subjective changes after contract defects are resolved."    correction: "Stop once contract checks pass and list remaining subjective options as optional notes."  comparison_sprawl:    trigger: "The comparison loop creates multiple final prompt systems instead of one selected output."    correction: "Select one primary output and move alternatives to fallback notes only."  execution_boundary_violation:    trigger: "The plan_execute_recap loop attempts project execution when the skill scope only allows planning or prompt generation."    correction: "Replace execution with execution instructions or a recap template."  capture_block_drift:    trigger: "The loop adds mandatory machine-readable capture blocks to every prompt body."    correction: "Move capture behavior into light_capture_hints or prompt_result_feedback."  provider_style_leak:    trigger: "The loop file starts defining provider-specific style rules."    correction: "Remove provider-specific style and reference provider style contract files."
```
