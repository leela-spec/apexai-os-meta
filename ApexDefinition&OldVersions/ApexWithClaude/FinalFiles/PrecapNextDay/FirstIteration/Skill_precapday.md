## Procedure

1. **Load best available context.** Read all supplied inputs, treat every missing input as a review flag instead of a blocker, and select the strongest available execution mode from full context, standard, recap recovery, bootstrap, calendar constrained, or prompt heavy.
2. **Create the day frame.** Generate `next_day_plan` metadata, input source status, day strategy summary, review status, and operator review flags using `low_confidence_auto_generated` when context is thin or bootstrap mode is active.
3. **Define fixed flow coverage.** Create or explicitly omit F1 Leela, F2 MasterOfArts, F3 Apex, and F4 Residual flow entries, including flow goal, sprint structure, expected outputs, compression reason, or omission reason with recovery logic.
4. **Create flow packets.** For each represented flow, define a `flow_packet` with sprint goals, expected outputs, raw_flow_dump preparation, skipped_flow_marker preparation, prompt_pack reference, and FlowRecap_handoff_block.
5. **Create prompt packs.** For each represented flow, generate or define one `flow_prompt_pack` containing finalized copy-paste prompts, prompt design rationale, provider rationale, workflow and process labels when useful, expected_output_type, failure hints, light capture hints, and usage_tracking_tags.
6. **Apply dependency interfaces.** Use prompt-engineering, usage-tracking, and workflow-process validation inputs when available; when a dependency is missing, run in degraded mode, mark the relevant confidence or review flag, and do not hard-block plan generation.
7. **Prepare calendar workflow-block writes.** Read calendar events or manual calendar constraints when available, create calendar_event_write_request or calendar_event_update_request only for workflow blocks, and require operator acceptance before any actual calendar write or update.
8. **Validate and present.** Check that the plan exists, every represented flow has a packet and prompt-pack reference, omitted flows have reasons, calendar writes are request-based, usage tracking hooks exist, FlowRecap handoffs exist, and all unresolved uncertainty is captured as operator_review_flags.
