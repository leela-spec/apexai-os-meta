# Managed Area

This area holds managed runtime files that define base governance, configuration, and repeatable rituals.

- `config/` contains runtime configuration.
- `rules/` contains managed governance that complements user-owned overrides.
- `rituals/` contains startup and maintenance procedures.
- `agents/` contains named seed agents, index surfaces, and durable overlap boundaries.
- `knowledge/` contains KB-lane ownership, starting-source maps, promotion ledgers, cross references, and overlap matrices.
- `processes/` contains orchestration flow, handoff contracts, and bounded research-to-patchspec workflows.

Managed files should change deliberately and stay aligned with the frozen system structure.

`NewFinals/`, `BaselinePatches/`, and `AdvancedUpdateProcess/` are not managed runtime folders and must not be treated as managed authority.
