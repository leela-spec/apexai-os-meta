# Checklists Ritual

Use this file as a lightweight, operator-facing checklist surface.

For hygiene authority, use `managed/rules/QA_HYGIENE_PROTOCOL.md`.
This file does not define severity semantics, finding classes, finding-record minimums, routing consequences, or closure law.

## Foundation Checklist

- governance files present
- runtime base scaffold present
- validation lane present
- migration ledger present
- project subtree exists for OpenClaw

## Environment Checklist

- git fetch verified
- git push workflow verified when needed
- Python command-name test run
- Python absolute-path fallback tested if command-name test fails

## Safety Checklist

- do not assume legacy root content is canonical
- do not enable advanced orchestration before validation results exist
- do not rely on ambient Python in CI

## Interface Checklist

- required control surfaces present and readable
- required pointers resolve cleanly
- bounded entry is possible without broad tree traversal

## Trace Checklist

- recent bounded trace exists for active work
- current state is traceable to visible session output
- required trace is not missing for material changes

## Escalation Checklist

- blocking hygiene issues are surfaced explicitly
- hold or degraded-mode need is not left implicit
- unsafe continuation is not treated as normal progress

## Promotion Checklist

- truth changes are routed through the promotion path
- no accepted-truth mutation is being treated as routine state update
- promotion-related pointers and packet traces are visible when needed
